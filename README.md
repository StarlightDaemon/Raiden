# RAIDEN

RAIDEN is the canonical central toolkit/framework repository for reusable AI-agent governance, repository-structure, and execution-support artifacts.

## Current Status

- Canonical root scaffold established
- Canonical operating-intent layer established
- Initial canonical `toolkit/` subtree materialized
- Historical prototype material retained under `Source_info/`
- Reviewed prototype evidence retained under `reference-repos/`
- Canonical naming stack established:
  - `RAIDEN` = central governing agent/framework authority
  - `RAIDEN Instance` = downstream deployed repo-local form
  - `Edict` = managed core artifact within a `RAIDEN Instance`

## Repository Areas

- `Source_info/` - preserved source history from early prototype conversations and drafts
- `reference-repos/` - imported prototype repo snapshots for review
- `reference-reviews/` - per-repo assessments, import candidates, and cross-repo comparison work
- `toolkit/` - canonical central toolkit subtree for shared prompt assets and the first managed `Edict` package surface
- `OPERATING_INTENT.md` - canonical mission-and-operating-intent layer for resource-constrained RAIDEN development
- Root `.md` files - current canonical RAIDEN navigation, state, decisions, and intake control
- Artifact audience and intended-role classification are defined in `ARTIFACT_AUDIENCE.md`

## Working Rule

Imported prototype repos are evidence sources, not RAIDEN authority. Promotion into canonical RAIDEN artifacts happens only after review, comparison, and an explicit decision entry.
